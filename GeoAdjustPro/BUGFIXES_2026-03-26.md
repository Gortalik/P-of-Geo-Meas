## Исправленные ошибки

### 1. AttributeError: 'PointsTableView' object has no attribute 'update_data'
**Файл:** `GeoAdjustPro/src/geoadjust/gui/widgets/points_table.py`

**Проблема:** Метод `_refresh_data_views()` в главном окне вызывал несуществующий метод `update_data()` для таблицы пунктов.

**Решение:** Добавлен метод `update_data()` как алиас для существующего метода `load_from_data()`:
```python
def update_data(self, points: list):
    """Обновление данных таблицы (алиас для load_from_data)"""
    self.load_from_data(points)
```

**Результат:** Пункты теперь корректно отображаются в таблице после добавления.

```python
layout.addItem(spacer)
```

### 4. Отсутствие логирования в GUI
**Файл:** `GeoAdjustPro/src/geoadjust/gui/components/log_widget.py`

**Проблема:** Логи Python не отображались в виджете журнала приложения.

**Решение:** 
1. Создан класс `QtLogHandler` для перехвата логов Python:
```python
class QtLogHandler(logging.Handler, QObject):
    """Обработчик логов для Qt виджета"""
    log_signal = pyqtSignal(str, str)  # message, level
    
    def emit(self, record):
        msg = self.format(record)
        level = record.levelname
        self.log_signal.emit(msg, level)
```

2. Добавлена автоматическая настройка логирования в `LogWidget`:
```python
def _setup_logging(self):
    """Настройка перехвата логов Python"""
    self.log_handler = QtLogHandler()
    self.log_handler.log_signal.connect(self._handle_log_message)
    root_logger = logging.getLogger()
    root_logger.addHandler(self.log_handler)
```

**Результат:** Все логи Python теперь отображаются в виджете журнала в реальном времени с цветовой кодировкой по уровням (INFO, WARNING, ERROR).

**Результат:** Иконки теперь не накладываются друг на друга, интерфейс выглядит аккуратнее.
