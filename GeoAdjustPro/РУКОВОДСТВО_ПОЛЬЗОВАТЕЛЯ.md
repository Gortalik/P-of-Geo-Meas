## Начало работы

### Запуск программы

1. Откройте терминал в папке `GeoAdjustPro/src`
2. Выполните команду:
   ```bash
   python -m geoadjust
   ```

### Первый запуск

При первом запуске откроется приветственное окно с опциями:
- **Создать новый проект** - начать работу с нуля
- **Открыть существующий проект** - продолжить работу
- **Открыть последний проект** - быстрый доступ к недавним проектам

## Создание проекта

### Шаг 1: Запуск мастера создания проекта

**Меню:** `Файл → Создать проект` или `Ctrl+N`

### Шаг 2: Основная информация

Заполните поля:
- **Название проекта**: Уникальное имя проекта (например, "Съёмка участка №5")
- **Описание**: Краткое описание работ
- **Автор**: Ваше имя
- **Организация**: Название организации
- **Путь к проекту**: Папка для сохранения (будет создана папка с расширением .gad)

### Шаг 3: Система координат

Выберите систему координат:
- МСК-33, МСК-50, МСК-63 (местные системы)
- WGS-84 (глобальная система)
- ПЗ-90 (российская система)
- Или настройте пользовательскую СК

### Шаг 4: Параметры уравнивания

Настройте параметры:
- **Метод уравнивания**: Классический МНК или Робастный
- **Доверительная вероятность**: 0.95 (стандартно)
- **Максимальное число итераций**: 50
- **Точность сходимости**: 0.0001 м

### Шаг 5: Завершение

Нажмите **"Готово"** - проект будет создан и открыт.

## Работа с пунктами

### Добавление нового пункта

**Меню:** `Редактирование → Добавить пункт` или `Ctrl+P`

#### Вкладка "Основные"
1. **Название**: Введите название пункта (например, "ПЗ1", "RP1", "101")
2. **Тип пункта**: Выберите тип:
   - **Исходный (фиксированный)** - координаты известны и не изменяются
   - **Определяемый (свободный)** - координаты будут вычислены
   - **Приближенный** - есть приближенные координаты
3. **Описание**: Опишите местоположение (например, "Угол здания")
4. **Класс**: Выберите класс точности (1-4 класс, 1-2 разряд, съемочная)

#### Вкладка "Координаты"
1. **X (Север)**: Введите координату X в метрах
2. **Y (Восток)**: Введите координату Y в метрах
3. **H (высота)**: Введите высоту в метрах
4. Отметьте **"Высота определена"** если высота известна

#### Вкладка "Точность"
1. **mX**: СКО координаты X (по умолчанию 0.01 м)
2. **mY**: СКО координаты Y (по умолчанию 0.01 м)
3. **mH**: СКО высоты (по умолчанию 0.01 м)

### Редактирование пункта

1. Выберите пункт в таблице "Пункты ПВО"
2. Дважды щелкните или нажмите правую кнопку мыши → "Редактировать"
3. Внесите изменения
4. Нажмите **"OK"**

### Удаление пункта

1. Выберите пункт в таблице
2. Нажмите правую кнопку мыши → "Удалить"
3. Подтвердите удаление

## Работа с измерениями

### Добавление нового измерения

**Меню:** `Редактирование → Добавить измерение` или `Ctrl+M`

#### Вкладка "Основные"
1. **Тип измерения**: Выберите тип:
   - Горизонтальное направление
   - Горизонтальный угол
   - Наклонное расстояние
   - Горизонтальное расстояние
   - Превышение
   - Зенитное расстояние
   - Вертикальный угол

2. **Станция**: Выберите пункт, с которого выполнено измерение
3. **Цель**: Выберите пункт, на который выполнено измерение
4. **Цель 2**: (только для углов) Выберите второй пункт
5. **Номер приёма**: Укажите номер приёма (1, 2, 3...)

#### Вкладка "Значения"

**Для угловых измерений:**
- **Градусы**: 0-359
- **Минуты**: 0-59
- **Секунды**: 0-59.9999
- **Десятичные градусы**: Автоматически пересчитывается

**Для линейных измерений:**
- **Расстояние**: Введите расстояние в метрах

**Для превышений:**
- **Превышение**: Введите превышение в метрах (может быть отрицательным)

#### Вкладка "Точность"
1. **СКО измерения**: 
   - Для углов: в секундах (например, 5")
   - Для расстояний: в метрах (например, 0.005 м)
2. **Вес**: Вес измерения (по умолчанию 1.0)
3. **Автоматический расчет веса**: Отметьте для автоматического расчета

#### Вкладка "Дополнительно"
1. **Высота прибора**: Высота инструмента над пунктом (м)
2. **Высота цели**: Высота отражателя/марки (м)
3. **Температура**: Температура воздуха (°C)
4. **Давление**: Атмосферное давление (мбар)
5. **Примечание**: Дополнительная информация

### Редактирование измерения

1. Выберите измерение в таблице "Измерения"
2. Дважды щелкните или нажмите правую кнопку мыши → "Редактировать"
3. Внесите изменения
4. Нажмите **"OK"**

### Удаление измерения

1. Выберите измерение в таблице
2. Нажмите правую кнопку мыши → "Удалить"
3. Подтвердите удаление

## Уравнивание

### Классический МНК

**Меню:** `Обработка → Классический МНК`

**Метод:**
- Параметрический метод наименьших квадратов
- Минимизация суммы квадратов поправок

**Процесс:**
1. Построение системы уравнений
2. Расчет весов измерений
3. Решение нормальных уравнений
4. Вычисление поправок
5. Оценка точности

**Результат:**
- Уравненные координаты пунктов
- Поправки к измерениям
- СКО единицы веса (μ₀)
- Эллипсы ошибок

### Робастное уравнивание

**Меню:** `Обработка → Робастное уравнивание`

**Метод:**
- Устойчивое к грубым ошибкам
- Автоматическое понижение весов подозрительных измерений

**Преимущества:**
- Не требует предварительной очистки от грубых ошибок
- Более надёжные результаты при наличии выбросов

**Процесс:**
- Аналогичен классическому МНК
- Дополнительно: итеративное переопределение весов

## Анализ результатов

### Анализ по Баарду

**Меню:** `Анализ → Анализ по Баарду`

**Требование:** Выполненное уравнивание

**Результаты:**
- **Внутренняя надёжность** - способность обнаружить грубую ошибку
- **Внешняя надёжность** - влияние необнаруженной ошибки на результат
- **Список подозрительных измерений**

**Интерпретация:**
- Высокая надёжность (>0.8) - хорошо
- Средняя надёжность (0.5-0.8) - приемлемо
- Низкая надёжность (<0.5) - требуется усиление сети

### Поиск грубых ошибок

**Меню:** `Анализ → Поиск грубых ошибок`

**Требование:** Выполненное уравнивание

**Метод:**
- Статистический тест Тау
- Анализ нормированных поправок

**Результат:**
- Список измерений с грубыми ошибками
- Величина ошибки
- Рекомендации по исправлению

### Визуализация эллипсов ошибок

**Меню:** `Анализ → Эллипсы ошибок`

**Требование:** Выполненное уравнивание

**Отображение:**
- Эллипсы ошибок для каждого пункта
- Масштабированы для наглядности (×100)
- Цветовая кодировка по точности

**Использование:**
- Оценка точности определения пунктов
- Выявление слабых мест сети
- Планирование дополнительных измерений

### Тепловые карты

**Меню:** `Анализ → Тепловые карты`

**Требование:** Выполненное уравнивание

**Отображение:**
- Интерполированная карта точности
- Цветовая шкала (зелёный - хорошо, красный - плохо)

**Использование:**
- Общая оценка качества сети
- Визуализация зон с разной точностью

## Советы и рекомендации

### Организация работы

1. **Создавайте отдельный проект для каждого объекта**
   - Легче управлять
   - Проще архивировать

2. **Используйте осмысленные названия пунктов**
   - ПЗ1, ПЗ2 - пункты закрепления
   - RP1, RP2 - реперы
   - 101, 102 - съемочные точки

3. **Регулярно сохраняйте проект**
   - Включите автосохранение
   - Делайте резервные копии

### Импорт данных

1. **Проверяйте данные перед импортом**
   - Используйте предпросмотр
   - Проверьте статистику

2. **Импортируйте поэтапно**
   - Сначала пункты
   - Затем измерения

### Уравнивание

1. **Всегда выполняйте предобработку**
   - Контроль допусков
   - Применение редукций

2. **Начинайте с классического МНК**
   - Если есть грубые ошибки - используйте робастное

3. **Анализируйте результаты**
   - Проверяйте μ₀ (должно быть близко к 1)
   - Смотрите на эллипсы ошибок
   - Ищите грубые ошибки

### Отчёты

1. **Формируйте отчёты после проверки результатов**
   - Убедитесь в корректности уравнивания
   - Проверьте все данные

2. **Сохраняйте отчёты в PDF**
   - Для архива и передачи заказчику

## Решение проблем

### Программа не запускается

**Проблема:** Ошибка при запуске

**Решение:**
1. Проверьте установку Python (версия 3.8+)
2. Установите зависимости: `pip install -r requirements.txt`
3. Проверьте путь к папке проекта

### Не импортируются данные

**Проблема:** Ошибка при импорте файла

**Решение:**
1. Проверьте формат файла
2. Убедитесь, что файл не повреждён
3. Проверьте кодировку (UTF-8 или Windows-1251)

### Ошибка при уравнивании

**Проблема:** Уравнивание не выполняется

**Решение:**
1. Проверьте наличие исходных пунктов
2. Убедитесь, что есть достаточно измерений
3. Проверьте связность сети
4. Выполните предобработку

### Большие поправки

**Проблема:** Поправки к измерениям слишком большие

**Решение:**
1. Проверьте исходные данные
2. Выполните поиск грубых ошибок
3. Используйте робастное уравнивание
4. Проверьте систему координат
